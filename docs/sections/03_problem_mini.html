Problem Definition

Time-series event retrieval asks: given a query segment, retrieve similar segments from an indexed archive. We frame this as similarity search by mapping fixed windows to embeddings and querying nearest neighbors in a vector database, rather than only doing classification. Retrieval quality depends heavily on the representation used before indexing, raw time windows, frequency views from FFT or STFT, or mixed time–frequency features, and these choices can behave differently across datasets and event types. We therefore compare representation and embedding strategies across seismic, physiological, and benchmark datasets to identify combinations that best preserve event similarity and transfer reliably across domains.
