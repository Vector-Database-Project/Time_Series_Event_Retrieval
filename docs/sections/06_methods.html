Methodology

Stage 1: Data and policy standardization. Datasets from the selected domains are collected, cleaned, and converted into a common analysis format using fixed-duration windows and a shared metadata schema. Metadata includes domain identifier, source or sequence ID for leakage-aware splits, operating condition when available, and optional labels. To preserve realistic evaluation conditions, splits are performed at the source level rather than at the random window level. This stage establishes a consistent preprocessing and evaluation policy across domains.

Stage 2: Encoding pipeline ablations and optimization of hyperparameters. This stage evaluates a 3 × 3 design across data representations and embedding methodologies to compare how different signal views interact with different learning strategies for retrieval.

Preprocessing and representation choices:

  • Time-only: raw standardized windows
  • Frequency-only: FFT and STFT based representations
  • Mixed time-frequency: combined temporal and spectral views

Embedding and learning strategies:

  • TS2Vec: self-supervised representation learning
  • InceptionTime: supervised learning based representation extraction
  • tsfresh: engineered-feature baseline without learned embeddings

Stage 3: Benchmarking, comparison, and conclusions. All configurations evaluated in Stage 2 are benchmarked and compared under a common retrieval evaluation protocol. This stage consolidates results to support final policy selection by comparing embedding methodologies, data representations, and the best-performing representation-method combinations within each domain. Final conclusions will be reported as comparative trends and policy recommendations, with emphasis on which combinations generalize most consistently across domains.
