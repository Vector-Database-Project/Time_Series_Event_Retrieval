Background

Time-series machine learning has traditionally focused on tasks such as classification, forecasting, anomaly detection, and segmentation. More recently, representation learning methods have made it possible to encode temporal data into compact vector embeddings that preserve useful structure for downstream tasks. These embeddings make it possible to treat event retrieval as a similarity search problem, where a query segment is matched against a database of previously indexed events.

This retrieval setting is useful in domains where analysts need to search large archives of temporal signals for recurring behaviors or related events. For example, seismic monitoring may require matching waveforms associated with similar earth events, while physiological monitoring may require retrieving similar cardiac or neural patterns from long recordings. A vector-database based retrieval pipeline can make this process scalable by storing learned representations and efficiently searching for nearest neighbors in the embedding space.

The main challenge is that retrieval quality depends strongly on how the time-series is represented before indexing. Raw time-domain signals, frequency-domain features, and mixed time-frequency representations may each preserve different aspects of event structure. Similarly, supervised, self-supervised, and non-learning baselines can produce embeddings with very different retrieval behavior. This motivates a comparative study of representation and embedding design choices for time-series event retrieval.
